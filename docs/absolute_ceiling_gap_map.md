# Absolute Ceiling Gap Ledger

## Purpose

This file is the live gap ledger for remaining ceiling blockers.
It is no longer a one-time historical analysis note.

## Open Gaps

### Reader Quality Gaps

- upper-tier heavy-reader contract is defined, but not every criterion is enforced with equally strong runtime evidence
- early-hook and episode-end compulsion still need stronger repeated unattended proof
- human-quality-lift still has not converged close enough to zero

### Story Machine Gaps

- relationship graph depth is still thinner than the target ceiling
- opponent-side planning and counter-strategy are still weaker than protagonist-side pressure
- scene-level realization checks against drafted output remain incomplete

### Autonomy And Runtime Gaps

- some final-threshold axes still depend on soak/fault evidence that is not guaranteed every unattended run
- append-only/control-loop proof still has false-negative risk in some success paths
- real API unattended validation is not yet as strong as stubbed validation

### Portfolio And Market Gaps

- market feedback loop exists, but business or reader response is not yet deeply steering next-generation policy
- portfolio allocation is stronger structurally than commercially validated
- long-horizon market adaptation remains lighter than the target end-state

### Policy And Operations Gaps

- document naming is still not normalized outside the core canonical stack
- some governance references still lag behind newly added canonical surfaces

### Business And Feedback Gaps

- business-side dashboards exist, but they are not yet tightly coupled to next-task generation
- revenue and campaign feedback are still weaker than direct story/runtime loops

## Closed Or Mostly Closed Gaps

- unified story-state architecture exists
- regression guard exists
- reward and serialization state exist
- reader-perception contract exists
- quality architecture and audit have canonical surfaces
- ops policy stubs were consolidated

## Prioritization Rule

Work the gaps in this order:

1. reader quality
2. autonomy/runtime
3. story machine
4. portfolio/market
5. business/feedback
6. naming and doc normalization

## Update Rule

When a gap materially changes:

- update this ledger
- update `WEBNOVEL_ALIGNMENT_STATUS.md`
- update the relevant canonical doc if the change affects authority
