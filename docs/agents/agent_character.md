# Agent Character

## Diagnosis

Character logic existed but was too flat.

## Ceiling Bottleneck

Without wound/lie/moral limit/obsession/contradiction/relationship debt, forced choices remain generic.

## Missing Architecture

- richer internal contradiction
- relationship debt and chemistry pressure
- ally axis, not only protagonist vs rival

## Proposed Subsystem

Extend cast and relationship state in `story_state_v2`, then derive prompt payload from that richer state.

## Integration Points

- conflict stakes
- information/emotion buildup
- chemistry-aware retention

## Regression Risks

- overloading prompts with too much metadata
- melodrama inflation if chemistry is not bounded

## Test Plan

- pressure increases decision pressure
- event fallout mutates progress/backlash
- relationship payload exposes usable chemistry/debt signals
