# Agent Event

## Diagnosis

Typed event logic existed but was narrow and still partly periodic.

## Ceiling Bottleneck

Without richer event types, serialization becomes repetitive and predictable.

## Missing Architecture

- sacrifice
- timer
- power_shift
- false_victory
- collapse
- misunderstanding

## Proposed Subsystem

Expand typed event slate and choose from state pressure, instability, reward balance, and chemistry pressure.

## Integration Points

- cliffhanger mode selection
- world change engine
- reward/serialization engine

## Regression Risks

- event diversity without causality
- too many exotic beats damaging coherence

## Test Plan

- known states select expected event families
- event payload carries irreversible cost and carryover pressure
